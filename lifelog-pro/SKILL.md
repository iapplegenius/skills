## Phase 1 · 自动采集数据

**规则：在询问用户手动提供任何内容之前，先运行自动采集脚本。**

```bash
python3 /tmp/lifelog-pro/scripts/collect.py
```

输出文件：`/tmp/lifelog-collect/collected_data.json`

采集内容（跨平台，任何一项失败自动跳过，不中断流程）：
- Shell history（bash/zsh/fish/PowerShell）
- AI Agent 对话记录（Claude Code/Cursor/Codex/Hermes）
- 浏览器历史（Chrome/Firefox/Edge/Safari）
- 软件/进程使用快照
- 命令时间间隔序列（用于能量曲线构建）
- 今日编辑文件列表
- 过去30天历史日记

**采集结束后**：向用户汇报哪些数据源采集成功/失败，然后继续执行 Phase 2。用户额外手动提供的内容与自动采集结果合并处理。

## Phase 2 · 解析数据

### 2a · Shell History + 二阶信号

**基础解析**（必须执行）：
- 命令分类：开发 / 运维 / 文件 / 网络 / 数据 / AI工具 / 其他
- 提取项目名（从路径、git仓库名、pip包名推断）
- 提取关键操作（git commit 消息、安装的包、创建的文件）
- 识别错误与重试（连续失败命令）

**二阶信号提取**（有时间戳时必须执行，无时间戳时跳过）：

```
时间间隔分析：
- 命令密度：每小时命令数 → 输出能量曲线数据点
- 专注块识别：连续15分钟内无 >5分钟中断 = 一个深度专注块
- 恢复速度：错误发生后多久出现成功命令
- 搜索行为：浏览器同一问题域的搜索次数（知识盲区深度）
- 工具切换频率：IDE↔浏览器↔聊天的切换次数（分心指数）

无时间戳时：以命令顺序估算相对密度，所有时间列填"—"，标注"时间估算"
```

### 2b · AI Agent 对话解析

从采集的 agent 日志中提取：
- 对话时长与轮次
- 涉及的项目/任务
- 卡点识别（反复修正、同一问题多次询问）
- 使用的 agent 类型

### 2c · 浏览器历史解析

- 分类：技术文档 / AI工具 / 社交 / 视频 / 购物 / 新闻 / 其他
- 从 Google/Bing URL 的 `q=` 参数提取搜索关键词
- 同一问题的重复搜索次数 → 知识盲区信号
- **隐私过滤**：银行/支付/登录页面 → `[已隐去敏感URL]`

### 2d · 软件使用 + 能量曲线构建

从进程快照 + 命令时间戳合并，构建以下结构（后续 SVG 生成依赖此数据）：

```json
{
  "energy_curve": [
    {"hour": 9, "density": 0.8, "type": "dev"},
    {"hour": 14, "density": 0.2, "type": "browser"}
  ],
  "focus_blocks": [
    {"start": "09:15", "end": "11:30", "duration_min": 135, "type": "deep_work"}
  ]
}
```

### 隐私过滤（全局规则）

含 `password / secret / token / key / api_key` 的命令 → `[已隐去敏感内容]`

## Phase 3 · 历史关联与拐点分析

> 详细算法见 `references/history_reader.md`（**必须在此阶段阅读**）

### 扫描路径（按优先级顺序）

```
1. /mnt/user-data/outputs/daily_*.md
2. ~/lifelog/daily_*.md
3. ~/Documents/lifelog/daily_*.md
4. ~/Desktop/lifelog/daily_*.md
5. ./daily_*.md
```

### 拐点识别（核心规则）

**必须输出具体数据和日期，禁止泛泛而谈：**

```
专注度拐点：
- 计算过去7天每日专注块数量
- 今天比前3天均值低30%以上 → 输出"专注度下滑拐点"
- 格式：你的专注度从[具体日期]开始下滑，那天你[具体行为]

项目停滞拐点：
- 某项目上次出现在日志 >7天 → "停滞警告"
- >14天 → "搁置警告"

技能短板复现：
- 同一短板历史日志出现3次以上 → "持续短板，建议专项解决"
- 本周首次出现 → "新发现短板"

正向连续信号：
- 连续N天有深度专注块 → "你已连续N天保持深度工作状态"
```

**输出示例（必须遵守此格式）：**
- ✅ `你已连续4天推进 project-x，上次停滞超过7天是在5月12日`
- ✅ `你在 Docker 权限问题上已经卡了3次（4/20, 5/3, 今天），这次该彻底解决了`
- ❌ `你最近一直在努力工作，继续加油`（禁止此类空话）

## Phase 4 · 实时新闻抓取

**必须使用 `web_search` 工具执行以下搜索，结果传给 Phase 5。**

```
搜索词（每项执行1次搜索，取最新且高质量的1-2条）：

- 新闻日期必须是今天或本周内（过滤过期内容）
```

## Phase 5 · 专家委员会

**这是本 skill 最核心的模块，必须生成五位专家的完整输出。**

### 强制规则（每位专家必须满足）

每位专家的输出**必须同时满足以下4条**，缺一不可：
1. **引用至少一条用户数据的具体细节**（命令名/URL/项目名/具体时间）
2. **引用至少一条今日新闻的具体事件**（新闻标题或事件名）
3. **给出一个具体可执行的行动**（格式："今天用X工具完成Y这件事"，不是"多学习"）
4. **字数控制**：每位专家正文 4-6 句话；综合专家正文 6-8 句话

### 五个专家角色与输出要求

#### 🔬 科技领域专家
- **身份**：顶级 LLM + 硅谷一线工程师视角
- **信息来源**：Phase 4 科技/AI新闻 + 用户开发行为数据
- **输出重点**：用户技术栈在行业的位置 → 行业动态影响 → 一个具体技术升级建议

#### 💰 商业/投资专家
- **身份**：顶级 LLM + 一线投资人/分析师视角
- **信息来源**：Phase 4 财经新闻 + 用户行为数据
- **输出重点**：用户技能/项目市场价值 → 宏观经济信号影响 → 职业/投资建议 + 风险提示

#### 🧬 学术/科学专家
- **身份**：顶级 LLM + 跨学科研究者视角
- **信息来源**：Phase 4 科研新闻 + 用户浏览器学习行为
- **输出重点**：用户学习内容在学术前沿的位置 → 相关科研发现 → 知识体系升级建议

#### 🧠 心理/健康专家
- **身份**：顶级 LLM + 认知科学/功能医学视角
- **信息来源**：Phase 2d 能量曲线/分心指数 + Phase 4 健康新闻
- **输出重点**：今日能量曲线解读（高峰/低谷/拐点）→ 行为背后心理信号 → 今晚1-2个具体健康行动

#### 🌐 全知综合专家（压轴，必须最后生成）
- **身份**：整合以上四位 + 用户数据 + 新闻的超级视角
- **输出重点**：用一段话串联四个视角的最关键信号 → 今天最重要的一件事 → **明天的第一个行动**（具体时间点 + 具体操作）

## 技能增强模块规范

**每次生成日报时必须执行此模块**，从当天操作中识别技能短板。

### 短板信号识别（按优先级）

```
1. 错误重试：同一问题出现 2+ 次错误 → 🔴 基础知识缺失
2. 绕道解决：用复杂方式解决本可简单完成的问题 → 🟡 进阶短板
3. 遗漏基础操作：完成任务后才发现有更直接的命令 → 🔴 基础
4. 依赖复制粘贴不理解参数 → 🟡 进阶短板
5. 手动重复可自动化的操作 → 🟡 进阶短板
```

### 分级标准

```
🔴 基础能力不足
  判断：该操作是日常基本动作，正确做法只需 1~2 条命令
  输出：直接列出"立即解决"的具体命令 + 简短解释

🟡 进阶能力短板
  判断：当前方案能工作但不够优雅/安全/高效，需系统学习
  输出：列出 2~3 个具体学习方向（工具名/概念名/文档方向）
```
