## Phase 1 · 自动采集数据

**在询问用户手动提供任何内容之前，先运行自动采集脚本。**

```bash
python3 /tmp/lifelog-pro/scripts/collect.py
```

输出：`/tmp/lifelog-collect/collected_data.json`

采集内容（跨平台，失败自动跳过）：
- Shell history（bash/zsh/fish/PowerShell）
- AI Agent 对话记录（Claude Code/Cursor/Codex/Hermes）
- 浏览器历史（Chrome/Firefox/Edge/Safari）
- 软件/进程使用快照
- **命令时间间隔序列**（二阶信号，用于能量曲线）
- 今日编辑文件列表
- 过去30天历史日记

采集完成后汇报哪些成功/失败，然后继续。
用户额外手动提供的内容与自动采集结果合并处理。

## Phase 2 · 解析数据

### 2a · Shell History + 二阶信号

基础解析（继承原版）：
- 命令分类：开发/运维/文件/网络/数据/AI工具/其他
- 提取项目名、关键操作、git commit消息
- 错误与重试识别

**新增·二阶信号提取：**
```
时间间隔分析（需带时间戳的history）：
- 命令密度：每小时命令数 → 绘制能量曲线
- 专注块识别：连续15分钟内无>5分钟中断 = 一个深度专注块
- 恢复速度：错误发生后多久出现成功命令
- 搜索行为：浏览器中同一问题域的搜索次数（知识盲区深度）
- 工具切换频率：IDE↔浏览器↔聊天的切换次数（分心指数）

无时间戳时：以命令顺序估算相对密度，标注"时间估算"
```

### 2b · AI Agent 对话解析

从采集到的 agent 日志中提取：
- 对话时长与轮次
- 涉及的项目/任务
- 卡点识别（反复修正、同一问题多次询问）
- 使用的 agent 类型

### 2c · 浏览器历史解析

- 分类：技术文档/AI工具/社交/视频/购物/新闻/其他
- 识别学习行为 vs 娱乐行为
- **搜索关键词提取**（从 Google/Bing URL 的 q= 参数）
- 同一问题的重复搜索次数 → 知识盲区信号
- 敏感URL过滤（银行/支付/登录页面）

### 2d · 软件使用 + 能量曲线构建

从进程快照 + 命令时间戳合并，构建：
```
能量曲线数据点：
[{hour: 9, density: 0.8, type: "dev"}, {hour: 14, density: 0.2, type: "browser"}, ...]

专注块列表：
[{start: "09:15", end: "11:30", duration_min: 135, type: "deep_work"}, ...]
```

### 隐私过滤

含 password/secret/token/key/api_key 的命令 → `[已隐去敏感内容]`
银行/支付/登录URL → `[已隐去敏感URL]`

## Phase 3 · 历史关联与拐点分析

详细策略见 `references/history_reader.md`

### 扫描路径（优先级）
```
1. /mnt/user-data/outputs/daily_*.md
2. ~/lifelog/daily_*.md
3. ~/Documents/lifelog/daily_*.md
4. ~/Desktop/lifelog/daily_*.md
5. ./daily_*.md
```

### 拐点识别算法（核心升级）

不再只做陈述，而是找**趋势拐点**：

```
专注度拐点：
- 计算过去7天的每日专注块数量
- 若今天比前3天均值低30%以上 → 识别为"专注度下滑拐点"
- 找拐点前一天的关键行为差异 → 输出："你的专注度从[日期]开始下滑，
  那天你[具体行为]，是什么让状态发生了变化？"

项目停滞拐点：
- 某项目上次出现在日志是N天前
- N > 7天 → "停滞警告"
- N > 14天 → "搁置警告"

技能短板复现：
- 同一技能短板在历史日志中出现次数
- 出现3次以上 → "持续短板，建议专项解决"
- 本周首次出现 → "新发现短板"

正向连续信号：
- 连续N天有深度专注块 → "你已连续N天保持深度工作状态"
```

### 成长关联叙事

用自然语言输出，**必须包含具体数据和日期**，禁止泛泛而谈：
- ✅ "你已连续4天推进 `project-x`，上次停滞超过7天是在5月12日"
- ✅ "你在 Docker 权限问题上已经卡了3次（4/20, 5/3, 今天），这次该彻底解决了"
- ❌ "你最近一直在努力工作，继续加油"

## Phase 4 · 实时新闻抓取

**使用 web_search 工具抓取当天的真实新闻，为专家委员会提供信息基础。**

按以下维度各搜索1-2条最新且高质量的信息：

```
搜索词列表：
1. 科技/AI动态：  "AI LLM latest news {今日日期}" + "tech industry today"
2. 财经/市场：    "Nasdaq stock market today" + "AI industry investment news"
3. 科学/学术：    "science research breakthrough this week"
4. 健康/医学：    "health wellness research news"

提取内容：
- 标题 + 一句话摘要 + 来源
- 与用户当天行为的关联点（比如用户用了Python，科技新闻里有Python相关）
- 新闻日期必须是今天或本周内（过滤过期内容）
```

搜索结果传给 Phase 5 的专家委员会作为"时事背景"。

## Phase 5 · 专家委员会

**这是本 skill 最核心的模块。**

基于用户当天数据摘要 + 历史分析 + 实时新闻，以四个领域专家视角输出建议，最后由综合专家汇总。

### 强制规则（防止模板化）

每位专家的输出**必须**：
1. **引用至少一条来自用户数据的具体细节**（命令、URL、项目名、时间）
2. **引用至少一条来自今日新闻的具体事件**（标题/事件名）
3. **给出一个具体可执行的行动**，不是"多学习"，是"今天用X工具做Y这件事"
4. **字数控制**：每位专家 4-6 句话，综合专家 6-8 句话

### 五个专家角色

#### 🔬 科技领域专家
**身份**：顶级 LLM + 硅谷一线工程师视角
**信息来源**：今日科技/AI新闻 + 用户的开发行为数据
**输出重点**：
- 你今天用的技术/工具在行业里的位置
- 行业最新动态对你当前工作的影响
- 一个具体的技术升级建议（基于新闻 + 你的代码行为）

示例逻辑：
> "你今天花了2小时写 Markdown 文档。行业最新讨论（引用新闻）表明 HTML-first 的内容格式正在成为 AI 时代的新标准，因为 LLM 对 HTML 的解析质量显著优于 MD。建议你把 [具体项目] 的文档格式迁移到 HTML，这不是小事，是方向性问题。"

#### 💰 商业/投资专家
**身份**：顶级 LLM + 一线投资人/分析师视角
**信息来源**：今日财经新闻（纳斯达克、产业链、AI投资动态）+ 用户行为
**输出重点**：
- 用户当前技能/项目的市场价值判断
- 今日宏观经济信号对用户职业/项目的影响
- 投资/职业方向建议 + 具体风险提示

示例逻辑：
> "纳斯达克今日上涨X%，AI基础设施（铜/稀土/电力）需求信号强烈（引用新闻）。你今天在做的 [项目] 涉及 AI 推理部署，这个方向的市场窗口未来18个月可能是最大的。但你的技术栈里缺少云成本优化能力，这在接下来的商业竞争中会是一个实际的劣势。"

#### 🧬 学术/科学专家
**身份**：顶级 LLM + 跨学科研究者视角
**信息来源**：本周科研突破新闻 + 用户的学习行为（浏览器历史、视频记录）
**输出重点**：
- 你今天学到的东西在学术前沿的位置
- 一个你可能不知道但与你的工作高度相关的科研发现
- 知识体系升级建议（基于你的当前技能盲区）

#### 🧠 心理/健康专家
**身份**：顶级 LLM + 认知科学/功能医学视角
**信息来源**：用户行为二阶信号（能量曲线、分心指数、工作时长）+ 健康新闻
**输出重点**：
- 今日能量曲线的解读（高峰/低谷/拐点）
- 行为背后的心理信号（基于具体数据，不是猜测）
- 今晚就能做的1-2个具体健康行动

#### 🌐 全知综合专家（压轴）
**身份**：整合以上四位 + 用户数据 + 新闻的超级视角
**输出重点**：
- 用一段话把四个视角的最关键信号串联起来
- 指出今天最重要的一件事（可能是意外发现）
- **明天的第一个行动**（具体、可执行、时间点）

示例逻辑：
> "综合来看，你今天技术上有产出但方向存在微偏——[科技专家信号]。市场在给你的方向发出强信号——[商业专家信号]。你的能量今天在下午2点明显下滑——[健康专家信号]。把这三件事放在一起，我的判断是：[一个整合性结论]。明天早上最先做的一件事是：[具体行动，例如'用30分钟把X文档从MD转HTML格式，测试LLM解析效果']。"

### HTML 专家委员会样式

每位专家一张卡片：
| 专家 | 主色 | 边框 | 图标 |
|------|------|------|------|
| 科技专家 | `#f0fdf4` | `#4ade80` | 🔬 |
| 商业投资 | `#fefce8` | `#fde047` | 💰 |
| 学术科学 | `#f0f9ff` | `#7dd3fc` | 🧬 |
| 心理健康 | `#fdf4ff` | `#d946ef` | 🧠 |
| 综合专家 | 深色 `#1a1814` | `#c0392b` | 🌐 |

综合专家卡片用深色背景全宽显示，视觉上突出压轴感。

每张卡片包含：
- 图标 + 专家身份标签
- "今日新闻参考"角标（显示引用的新闻标题，灰色小字）
- 正文（4-6句话）
- "明天行动"徽章（仅综合专家卡片）

## 周报模式

触发词："生成周报"、"本周总结"、"weekly report"等

### 数据来源

自动扫描并读取过去7天的日报文件：
```
daily_YYYY-MM-DD.md × 7份（存在的部分）
```

### 周报结构

```
周报结构：
├── 本周概览（一句话 + 关键数字汇总）
├── 能量曲线对比（7天叠加SVG图）
├── 项目进展汇总
│   ├── 每个项目本周累计时间
│   ├── 推进了多少 / 停滞了多少
│   └── 下周优先级建议
├── 技能成长矩阵
│   ├── 本周新出现的技能
│   ├── 持续强化的技能
│   └── 重复出现的短板（需专项解决）
├── 行为模式周分析
│   ├── 最高效的一天（数据支撑）
│   ├── 最低效的一天（原因推断）
│   ├── 娱乐/学习/开发时间周占比
│   └── 专注块总计 vs 上周对比
├── 本周新闻对个人的影响（实时抓取本周重要新闻）
├── 专家委员会·周度版
│   ├── 科技专家：本周行业变化 × 你的技术演进
│   ├── 商业专家：本周市场信号 × 你的职业方向
│   ├── 学术专家：本周科研突破 × 你的知识盲区
│   ├── 健康专家：本周行为模式 × 身心状态趋势
│   └── 综合专家：下周最重要的一件事
└── 下周行动计划（优先级排序，3-5条）
```

### 周报输出文件

```
/mnt/user-data/outputs/
├── weekly_YYYY-WNN.md    （WNN = 周数，如 W22）
└── weekly_YYYY-WNN.html
```

### 数据不足的处理

- 只有1-2天日报：基于现有数据生成，标注"数据不足，仅供参考"
- 某天日报缺失：在时间线上标注"[数据缺失]"，不影响其他天分析
